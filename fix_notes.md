# 問題分析: AI生成クリップがDB端とほぼ同じ

## 現在のgenerate_clip処理フロー:
1. ダウンロード（指定区間のみ）
2. Speech boundary調整（文の途中でカットしない）
3. 正確なカット（time_start → time_end）
4. Person detection（人がいないシーンを除去）
5. Silence removal（無音区間を除去: -35dB, 0.8秒以上）
6. Whisper文字起こし
7. GPT字幕リファイン
8. 1080x1920縦型クリップ作成（字幕なし - フロントで表示）
9. Hook挿入
10. SE挿入
11. アップロード

## 根本原因:
**カット処理が「無音除去」と「人物非検出シーン除去」しかない！**

- 無音除去: -35dB, 0.8秒以上 → ライブコマースでは常に話しているので無音がほぼない
- 人物検出: ライブコマースでは常に人が映っているので除去されるシーンがない
- **結果: 元のtime_start〜time_endの区間がそのまま出力される**

## DB端との違い:
- DB端 = 動画解析時にphaseごとに区切られたクリップ（time_start〜time_end）
- AI生成端 = 同じtime_start〜time_endを使って、字幕追加 + hook追加しただけ

## 本来あるべき処理:
1. **コンテンツベースのカット**: 「えーと」「あのー」等のフィラー除去
2. **テンポ調整**: 話の間を詰める（0.3-0.5秒の短い無音もカット）
3. **不要シーン除去**: 商品説明以外の雑談、挨拶等をカット
4. **ジャンプカット**: TikTok風の素早いカット切り替え

## 修正方針:
1. silence detection閾値を大幅に下げる: -35dB → -25dB, 0.8秒 → 0.3秒
2. フィラー検出+除去を追加（Whisper結果から「えーと」「あのー」等を検出）
3. GPTによるコンテンツ分析で不要区間を特定してカット
4. speed_factorのデフォルトを1.0 → 1.05に（微妙にテンポアップ）
